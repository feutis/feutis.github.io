2022-02-28

hexo blog搭建方案

搭建hexo blog的过程中，本来希望能够完全依托于wsl搭建，但是发现在linux下没有和typora类似的好用的编辑器，vscode的插件也用起来没有typora爽，同时wsl安装nvm还有奇怪的bug，每次都要进管理员账号进行使用，非常麻烦，所以放弃了wsl，在windows下搭建，本文主要记录了的是图床方案的解决和公式的显示，至于其他的安装部分会记录一些自己遇到的问题，至于nvm nodejs npm的下载，我的建议是直接下载nvm，然后通过nvm再去下载nodejs和npm，这样比较简单也方便，就是注意需要自己把npm模块的路径加到环境变量path里去。

hexo安装

下载nvm后通过nvm下载npm和nodejs，然后安装hexo

npm install hexo-cli@latest -g

进入自己的blog目录后（没有就新建一个）使用hexo init初始化blog。

以下方式已经弃用，建议直接使用windows下的nvm

先安装nvm，通过nvm管理npm和nodejs，否则会出现比较诡异的权限问题，哪怕使用sudo赋予权限也依旧会产生问题，官方推荐的解决方案就是使用nvm进行管理，具体问题与解决方案。

wsl中直接运行nvm官方推荐的安装方式也是不能够运行的，会有错误，具体可以参照这里，这里没有解决，直接加入-k参数忽略该问题即可。

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.1/install.sh | bash

但是这样依旧下载不下来，目测是权限问题，加了sudo也无法正常运行，wsl的权限真的很怪，没办法，直接

sudo -i

用管理员账号手动把install.sh下载下来，然后运行，成功安装nvm。但是这样每次都要进root账号才能使用npm，非常麻烦，已弃用，全作为记录留存。

picgo+gitee图床方案

gitee设置

为了能够在本地显示图片，需要一个地方用于存放我们的图片，这里选择gitee，新建一个gitee仓库，比如叫他img，注意该仓库需要设置为公开，现在在创建的时候好像已经不能设置为公开了，需要自己进入仓库，在设置里修改为公开。记录一下仓库名称，就是进入仓库后url的后面部分，比如我这里的仓库名称就是fffyutt/img，后面有用。

随后创建一个身份令牌，在gitee页面右上角头像处进入设置页面，在安全设置中选择私人令牌，点击右上角的生成，只需要给projects权限即可，把token复制下来后面有用。

picgo设置

在picgo官网下载picgo，要是找不到哪里下载直接下载这个，安装完成后打开，在插件设置里搜索gitee-uploader（这个插件用于让picgo支持gitee图库）然后下载，这里还需要下载一个compress插件（这个插件用于让picgo上传图片时自动压缩一下，压缩到1M以下，因为gitee1M以上的文件需要登陆查看，为了避免图片无法加载，上传前需要压缩至1M以下），可以先试试能不能直接下载，我直接下载会出问题，反正就是下载不成功，目测又是windows奇怪的权限问题，有评论指出是因为picgo的module安装在c盘内，问题描述和解决方案参考这里的answer6，我采用了别的方法解决。

可以查看picgo-plugin-compress的官方页面，里面推荐直接使用npm安装，但是实际上还是不能安装上去的，最后使用cnpm安装成功，注意要先进入picgo的安装目录比较好，这样不会把这个插件乱装在别的位置

npm install picgo-plugin-compress --save --registry=https://registry.npm.taobao.or

如果没有cnpm可以先安装cnpm

npm install cnpm -g

安装成功后在picgo插件设置右上角的load按键中将picgo-plugin-compress加入进来，具体位置在刚刚执行cnpm的目录的node_modules下，有一个叫做picgo-plugin-compress快捷方式，选择这个就行了。

加入进来后点击picgo-plugin-compress下面的设置（小圆圈），选择启用插件（注意第一行和最后一行都有启用），然后点击配置picgo-plugin-compress，compress选项选择lubangitee，可以在官方git页面找到各个配置项都是什么意思。

在picgo的图床设置里找到gitee，设为默认图床，在repo中输入刚刚记录下来的仓库名，在token中填刚刚记录下来的token，path指的是在仓库内部的路径，一般填img就行，至于branch，没看懂的master就行，下面的customPath可以随便填，但是customUrl空着就行。我的设置如下。

至于picgo的设置里其他设置怎么设置，可以参考官网的介绍，我只打开了一个时间戳重命名。

typora设置

在typora中偏好设置内将图像（如果你没有typora建议下载一个，很香），上传服务选择picgo（app），并指定picgo路径，上面的是否自动上传就是个人喜好问题了。然后就点击验证图片，成功了！

hexo+katex显示公式

因为hexo原生的渲染器不支持公式，运行如下命令换一个支持渲染公式的渲染器

npm uninstall hexo-renderer-marked --save
npm install hexo-math --save
npm install hexo-renderer-markdown-it-plus --save

然后在_config.yml加入这些内容

math:
  engine: katex
  katex:
    css: https://cdn.jsdelivr.net/npm/katex@0.10.0/dist/katex.min.css
    js: https://cdn.jsdelivr.net/npm/katex@0.10.0/dist/katex.min.js
    config:
      # KaTeX config
      throwOnError: false
      errorColor: "#cc0000"

然后就可以使用

$$a=1$$ 用作行间公式，效果如下：

a=1a=1 a=1

$a=1$ 用作行内公式，效果如下：a=1a=1a=1

hexo常用插件

整理为package.json，直接把需要的内容加入到自己package.jison即可

{
  "name": "hexo-site",
  "version": "0.0.0",
  "private": true,
  "scripts": {
    "build": "hexo generate",
    "clean": "hexo clean",
    "deploy": "hexo deploy",
    "server": "hexo server"
  },
  "hexo": {
    "version": "6.0.0"
  },
  "dependencies": {
    "hexo": "^6.0.0",
    "hexo-cli": "^4.3.0",
    "hexo-generator-archive": "^1.0.0",
    "hexo-generator-category": "^1.0.0",
    "hexo-generator-index": "^2.0.0",
    "hexo-generator-tag": "^1.0.0",
    "hexo-math": "^4.0.0",
    "hexo-renderer-ejs": "^2.0.0",
    "hexo-renderer-markdown-it-plus": "^1.0.4",
    "hexo-renderer-pug": "^3.0.0",
    "hexo-renderer-stylus": "^2.0.1",
    "hexo-server": "^3.0.0",
    "hexo-theme-landscape": "^0.0.3",
    "hexo-theme-yun": "^1.8.10"
  }
}

或者使用如下命令安装

npm install --save hexo-cli 
npm install --save hexo-server 
npm install --save hexo-renderer-pug #因为hexo-render-pug会让npm报红，我很难受，所以换了这个
npm install --save hexo-renderer-stylus 
npm install --save hexo-math
