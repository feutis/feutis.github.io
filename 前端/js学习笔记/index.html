2021-04-02- 2022-02-25
前端-前端js

数据类型

主要包括原始类型和引用类型

原始类型声明后存入栈中，如果对已经声明并赋值的变量赋值，则在栈中重新选择一个空间存入该值

//假设我们声明并赋值a，目前的栈顶位置为0x100
var a = 3;   //此时会在0x100存入a的值3
a=4； //会在0x101位置重新存入a的值4

引用类型声明后在栈中开辟空间，赋值后在堆中开辟空间存入所赋的值，并将堆中的地址p存入栈中对应空间，此时定义内容相同的变量，那么新的变量会与原变量共享堆中的数据，即新变量在栈中开辟空间并指向堆中地址p（此时修改任一变量两个变量内容均修改）。

其实和java非常像

//栈顶0x100，堆顶0x200
var a = [1, 2, 3]; //在0x200位置存入[1, 2, 3],并将0x200存入0x100
var b = [1, 2, 3]; //将0x200存入0x100的下一个位置(假设为0x101)
//此时修改b会将a一起修改(指添加值删去值)

类型转换

显式类型转换

//以简单的例子说明问题
typeof(typeof(NaN)) //=string
typeof(Number("123")) //=number
typeof(undefined) //=NaN
typeof(null) //=0
Number("123") //=123

typeof(parseInt("12a34"))  //=12  取到第一个非数字

num.toFixed(3);  //=3.142 四舍五入
num.toString(16); //  暂时不知道

Boolean(x); //x = null undefined NaN "" 0 false时输出false,其余时刻为true

隐式类型转换

var a = "123";
a++;   //=124 转换为Number
var a = "a" + 1; //=a1 1转换为"1"
var a = "3" * 2; // * / - % 都会让str->number
var a = 1 > "2"; //转换为number  >= !=等同理
var a = "b" > "a"; //转换为ascii
var a = 1 == "1"; //=ture
var a = 1 === "1"; //=false 不进行隐式转换
//在字符串前后加上 + 或者 - 字符串变数字
isNaN("a") //=true 先把里面的东西转化为number再比较

类型的比较

var a = 1 == "1"; //=ture
var a = 1 === "1"; //=false 不进行隐式转换
var a = 2 > 1 > 3; //=false  ==> true > 3 ==> 1 > 3 ==> false
var a = NaN == NaN; //=false  NaN不等于任何数,包括他自己
// undefined和null既不大于0,也不小于0,也不等于0,但是undefined==null返回true,undefined === null返回false

函数

//格式
function name(){}

在函数内部，可以用arguments来操作调用函数时输入的参数，其中arguments是一个广义表，从0计数，可以采用arguments.length来获取输入的参数个数；同时，可以用函数名，如例子中的name来调用函数定义时括号内的参数（即，调用函数时输入的参数可以和函数定义时定义的参数数量不同，如果调用时输入的参数少了，多的参数被定义为undefined类型）

不妨假设函数的第一个参数为a，且在调用时被赋值了，我们需要注意到，虽然a与arguments[0]有着相同的值且能被一同修改，但是a存在栈中而arguments[0]存在堆中，他们不是一个东西，但在函数内部时有着映射关系。

函数的参数

形参与实参

形参：在函数声明时给出的参数列表中的值，在函数内部可以用与函数名同名的广义表访问，是一个占位符

实参：在函数调用时给出的参数列表中的值，在函数内部可以用广义表arguments访问

默认值

可以使用类似下式为函数形式参数赋予默认值（ES6之前的都不行）

function fact(n = 5, m){}

此时如果需要对m赋值但是保持n的默认值需要用undefined填充n的位置，如下

fact(undefined, 6); //n=5,n=6

对于ES5及之前，需要采用如下方式设置默认值

function fact(n, m){
	n = arguments[0] || 5; //n的默认值是5
}

作用域链

在js中，万物皆对象，包括所有的函数。而对象有一个我们不可访问的属性[[scope]]，用于存储作用域链的地址。

当windows被创建，js引擎会先按照GO的顺序将所有的定义在windows下的变量声明及函数声明存入GO，并将GO的地址存入windows.scope；当一个函数被丁一时，js引擎会创建函数的作用域链，将该链地址存入函数的scope，将GO存入作用域链的第一个位置，在函数被执行时，会按照AO的规则对AO进行初始化，并将AO的地址存入作用域链的表头，此时GO被挤到第二个位置了。此外，AO在函数执行完成后被销毁。上述具体流程如下图。

当js寻找一个变量时，会在作用域链中从上往下找，这也解释了局部变量会覆盖全局变量的问题。

闭包

当父对象结束时，父对象的AO还存在子对象的AO中，则子对象将父对象的AO带走了，父对象的AO并没有随着父对象的销毁而销毁。此时虽然父对象已 经结束，但是此时可以通过子对象调用父对象的变量与函数，因为此时父对象的变量与函数还在子对象的作用域链中。

当内部函数被返回到外部并保存时，一定会产生闭包，闭包急哦产生原来的作用域链不是放。

过度的闭包可能会导致内存泄漏，或者加载缓慢。

this

在对象初始化时被保存在AO/GO中，指向对象本身

在初始化对象时，如下写法相等。其实new就是创建了一个对象然后返回给了我们指定的变量，如果我们显示的返回this，可以不用new保留字，通过闭包的方法将对象返回。

function Car(){}
var car1 = new Car();
function Car(){
    return this
}
var car1 = Car();

当然你也可以在构造函数里直接return一个其他的在构造函数中的构造的对象，以此来覆盖默认返回的this对象（暂时还没研究函数返回相关的内容），如果直接 return; 后面不带任何内容，经过测试还是会返回this。

包装类

var a = 123;
a.len = 3; //此语句等价于 a = new Number(123).len = 3; delete a.len;
//即，先将a转化为对象，然后创建属性len=3，但是没有地方存放这个属性，所以直接删除（或者直接跳过存储），此时a已经不是对象了
console.obj(a.len); // 打印结果为undefined，先将a当作对象看待，但是a没有这个属性，所以返回undefined

对于字符串，本身没有length属性，但是在我们试图访问string的length属性时，和之前一样，系统试图将这个字符串转化为一个对象，最为合适的转化时String(str)，恰巧，new String()出来的对象是有length属性的（可以自己打印下试试），访问完length后这些值也被销毁了，但是访问成功了，已经打印出来了。下面两个式子等价

console.log(str.length);
console.log(new String(str).length);

所以，我们试图去改变一个原始类型string的length属性是不会成功的，因为他根本没有这个属性。但是这里可能有另外一种解释，在js中，字符串是不可变的，修改字符串也只是返回另外一个字符串的地址而非修改原来的同地址的字符串，这里面length和字符串长度的映射关系未知，有待研究，但是有一点可以确定，length是个只读属性，永远无法修改。

原型 prototype

在对象（这里叫他1）里有一个__proto__属性，这个属性是一个对象（这里叫他2），一个引用类型，其内容存在堆中，其指针存在对象1所对应的堆内存里。我们可以通过__对象1.prototype.内容__来修改对象2堆内存里的东西，如果此时对象1已经实例化，其__proto__依旧会被修改，但是如果通过__对象1.prototype = {}__对对象2进行修改，则会重写分配堆内存，如果在修改之前就对象1已经实例化，则对象1.__proto__不会被修改，可以理解为产生了闭包。

继承

call/apply

function Car(a, b){
	this.a = a;
	this.b = b;
}
var fcar = {
	c: "c"
};
Car.apply(fcar, ["a", "b"]);
Car.call(fcar, "a", "b");
//上面两句等价,第一个参数是将Car中的this重定向为fcar了,也就是会向fcar里面按照Car构造方法写入a和b
